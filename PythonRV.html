一、输入/输出与数据类型转换

# 核心现象 1: input() 吞进去的所有东西都是字符串
ans = input() # 即使键盘输入的是 100
print(type(ans)) # 必然输出 

# 核心现象 2: print() 的隐藏参数
print("A", "B", sep="-", end="*") # 输出 A-B* (不换行)

# 核心现象 3: 强制转换的边界
print(int(3.9))    # 截断取整输出 3 (不是四舍五入)
print(int("12"))   # 成功转换为整数 12
# int("3.14")      # 报错！字符串内含有小数点无法直接转int
  • 真题陷阱：直接执行 input() + 1 会发生 TypeError 报错，必须先加 int()。
  • 类型区别：3 是 int，3.0 是 float。

二、字符串格式化与高阶切片

pi = 3.1415926
# 现象 1: 精确控制小数位数 (自带四舍五入)
print(f"{pi:.2f}") # 输出: '3.14'
print(f"{pi:.4f}") # 输出: '3.1416'

# 现象 2: 整数占位补零
num = 7
print(f"{num:03d}") # 输出: '007' (总共3位宽度)

# 现象 3: 切片步长与逆序
s = "ABCDEFG"
print(s[1:5:2]) # 输出: 'BD' (索引1和3，不含5)
print(s[::-1])  # 输出: 'GFEDCBA' (经典字符串反转)
  • 致命概念：f"{pi:.2f}" 的结果是一个字符串，而不是数字！
  • 字符串是不可变对象。执行 s.replace('A','X') 后不赋值回 s，s 绝不改变。

三、四大算术除法与运算符优先级

# 1. 真除法 / -> 结果必为浮点数
print(6 / 2)   # 输出 3.0 (绝对不是整数 3)

# 2. 地板除 // -> 向下取整
print(7 // 3)  # 输出 2
print(-7 // 3) # 输出 -3 (向更小的负数方向靠拢)

# 3. 取余 % -> 常用于周期性与奇偶判断
print(7 % 3)   # 输出 1
print(4 % 2)   # 输出 0 (说明4是偶数)
  • 终极优先级：括号 () ➔ 乘方 ** ➔ 乘除整除取余 *, /, //, % ➔ 加减 +, - ➔ 比较运算符 ➔ 逻辑运算符（not ➔ and ➔ or）。

四、四大集合容器微观辨析

容器符号可重复可修改关键特性/高频考点
列表 (list)[]✅ 是✅ 是append()追加，pop()删除
元组 (tuple)()✅ 是❌ 否t=(1)是int！单元素必须写成(1,)
字典 (dict){}❌ 键无✅ 是键值对。键（key）必须不可变，列表不能作键
集合 (set){}❌ 否✅ 是自动去重。例如把列表转集合去重
  • 清空与删除：列表的 remove(x) 删除的是具体的值，而 del arr[0] 删除的是索引处对应的值。

五、条件分支控制与 range() 边界

# if-elif 的短路特性
score = 90
if score >= 60:
    print("及格")
elif score >= 80:
    print("优秀") # 永远不可能走到这里！因为被上面拦截了

# range(start, stop, step) 左闭右开
print(list(range(5)))     # [0, 1, 2, 3, 4] (默认从0开始)
print(list(range(1, 5)))  # [1, 2, 3, 4] (不含结束值5)
print(list(range(5, 1, -2))) # [5, 3] (负数步长逆序)
  • 缩进规则：
    • Python 强制依赖缩进。同一代码块的缩进空格数必须绝对一致，否则报 IndentationError。

六、全局与局部变量及异常处理

n = 10 # 全局变量
def change():
    global n # 必须声明，否则内部写 n=20 会变成局部变量
    n = 20

# try-except 捕捉精确错误
try:
    ans = 10 / 0
except ZeroDivisionError:
    print("抓到了！除数不能为0")
except TypeError:
    print("类型错误")
  • 变量作用域：函数内部可以直接**读取**全局变量的值，但若想**修改（重新赋值）**全局变量的值，必须在函数内部首行写 global 变量名。

七、二维列表与文件控制

# 二维列表的定位: [行][列]
grid = [
    [1, 2, 3], # 第 0 行
    [4, 5, 6]  # 第 1 行
]
print(grid[1][2]) # 定位第1行第2列 -> 6

# 文件读写三字诀: r(读), w(覆盖写), a(追加写)
with open("test.txt", "w") as f:
    f.write("Hello") # 自动关闭文件，防止内存泄露
  • 文件模式重难点："w" 模式极度危险，只要打开就会彻底清空原文件；想在后面加内容，必须用 "a" 模式。

八、常用标准模块 (random / time)

import random
import time

print(random.randint(1, 5)) # 产生[1,2,3,4,5]中的任意整数
# 注意: randint 极其特殊，它是【左闭右闭】的！

print(random.random()) # 产生 0.0 到 1.0 之间的随机浮点数

print(time.time()) # 返回当前的时间戳(大浮点数)
time.sleep(2)      # 整个程序强制静止（睡眠）2秒钟
  • 唯一特例：Python 里几乎所有的截取和生成都是左闭右开，唯独 random.randint(a, b) 能生成边界值 b！

九、大赛核心算法逻辑辨析

  • 枚举算法：俗称“死笨方法”，用 for 或 while 循环把所有可能的情况全部尝试一遍，符合条件的留下。
  • 模拟算法：题目说怎么做，代码就怎么原封不动地还原。例如：模拟时钟转动、模拟发牌。
  • 贪心算法：只看眼前。每一步都做出在当前看来最好的选择，从而期望得到全局最优解（如：人民币找零钱，优先给面值最大的）。
  • 递归算法：自己调用自己。核心考点是**必须有明确的终止条件**，否则会引发 RecursionError（死递归）。
